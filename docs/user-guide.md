# OpenClaw Installer 用户使用说明

## 1. 软件用途

OpenClaw Installer 是一个面向 macOS 普通用户的 OpenClaw 安装、配置和验证助手。

它的作用是把原本需要手动输入命令的流程，整理成更容易理解的图形界面操作：

- 检查电脑环境是否满足要求
- 安装 OpenClaw 本体
- 打开 OpenClaw 官方配置向导
- 验证 OpenClaw 是否已经可以基本使用
- 在遇到问题时查看安装日志

本工具不会自己保存 API Key，也不会自己写 OpenClaw 的官方配置文件。API Key、模型服务商、默认模型等内容由 OpenClaw 官方配置向导处理。

## 2. 推荐使用流程

建议第一次使用时按下面顺序操作：

1. 点击“开始检测”，确认当前电脑环境是否满足安装要求。
2. 点击“一键安装”，安装 OpenClaw 本体。
3. 点击“配置 API”，准备进入 OpenClaw 官方配置向导。
4. 先阅读 GUI 中显示的配置流程说明，了解接下来会遇到哪些问题。
5. 在打开的 Terminal 中，根据提示完成 OpenClaw 官方配置向导。
6. 配置完成后，回到本软件，点击“我已完成配置，立即验证”。
7. 如果安装或配置遇到问题，点击“查看日志”排查原因。

如果你不确定某一步该怎么选，优先选择默认项；遇到增强功能时，可以先跳过，等 OpenClaw 能正常使用后再慢慢调整。

## 3. 配置 API 时可能遇到的步骤

点击“配置 API”后，系统会打开 Terminal，并运行 OpenClaw 官方配置向导。你可能会看到下面这些步骤：

- 安全确认：个人使用一般选择 Yes。
- Setup mode：第一次使用建议选择 QuickStart。
- Model provider：选择你的 API 来源，例如 OpenRouter、DeepSeek、OpenAI 等。
- API Key：粘贴你自己的 API Key。
- Default model：如果不懂，可以保持默认。
- Channel：第一次体验建议选择 ClickClack。
- Web search / Skills / Hooks：如果不清楚用途，可以先选择 Skip for now。
- Gateway service：保持默认安装即可。
- 完成后：回到 OpenClaw Installer，点击“我已完成配置，立即验证”。

配置向导由 OpenClaw 官方命令完成，本工具只是帮你打开它，并在完成后引导你做验证。

## 4. 常见问题

### 为什么会打开 Terminal？

OpenClaw 官方配置向导是交互式命令行程序，需要用户选择选项并输入 API Key。Electron GUI 后台不适合静默处理这种交互，所以本工具会打开系统 Terminal，让你直接使用官方向导。

### API Key 会不会被本工具保存？

不会。本工具不保存、不读取、不上传你的 API Key。API Key 由 OpenClaw 官方配置流程处理。

### 安装时间较长怎么办？

安装过程需要访问网络并执行 OpenClaw 官方安装脚本。如果等待时间较长，请先不要关闭软件。若安装失败，可以点击“查看日志”查看详细原因。

### 配置完成后为什么还要验证？

Terminal 中的配置完成，只表示官方向导已经结束。是否真的可用，还需要通过“验证可用”确认 OpenClaw 命令、版本和配置文件路径是否正常。

### 日志在哪里？

正式执行“一键安装”后，安装日志会写入：

```text
~/.openclaw-installer/logs/
```

dry-run 或预演模式不会写日志。如果日志目录还不存在，说明你可能还没有执行过正式安装，或者安装尚未产生日志。

### 配置状态分别是什么意思？

- 待配置：还没有完成 OpenClaw 官方配置，或尚未检测到可用配置。
- 等待验证：已经打开配置向导，但还没有通过本软件验证。
- 已配置：验证时检测到 OpenClaw 基本可用，并且配置文件路径可读取。
- 配置异常：验证失败，或配置状态不完整，需要重新检查配置流程。
