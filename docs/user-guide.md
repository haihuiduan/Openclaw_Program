# OpenClaw 安装助手用户指南

## 1. 如何开始

OpenClaw 安装助手用于在 macOS 上准备 OpenClaw、配置 AI 服务商 API Key，并打开 OpenClaw 控制台开始使用。

首次使用建议按界面提示完成三件事：

1. 准备 OpenClaw
2. 配置 AI 服务商 API Key
3. 打开控制台开始使用

本工具不会保存、读取或上传你的 API Key，也不会自己修改 OpenClaw 的官方配置文件。配置会交给 OpenClaw 官方流程完成。

## 2. 准备 OpenClaw

点击“开始”后，进入“准备 OpenClaw”。

本步骤会检查这台 Mac 是否满足运行条件，并安装或确认 OpenClaw：

- 检查基础环境
- 检查是否已安装 OpenClaw
- 安装 OpenClaw
- 确认命令可用

如果 OpenClaw 已经安装，本工具会跳过重复安装。首次安装可能需要几分钟，请保持网络连接，不要关闭窗口。

如果缺少 Node.js、npm 或 Git，界面会给出处理建议。本工具不会自动安装系统级依赖，也不会偷偷修改 PATH。

## 3. 配置 API Key

在“配置 API Key”步骤中，选择你的 AI 服务商并粘贴 API Key。

当前普通流程支持：

- OpenRouter
- DeepSeek
- OpenAI
- Gemini
- Qwen

模型可以先选择“自动推荐，适合首次使用”。后续需要时，可以再更换模型或进入官方配置向导做更多设置。

API Key 只会用于提交给 OpenClaw 官方配置流程。本工具不会展示、保存或写入日志。

## 4. 打开控制台

配置检查通过后，点击“打开控制台”。

OpenClaw 控制台会在浏览器中打开。你可以在那里开始使用 OpenClaw，例如：

- 和 AI agent 对话
- 总结文件或资料
- 整理任务和计划
- 辅助写作
- 后续连接更多工具和聊天渠道

如果浏览器没有成功打开，可以回到本软件再次点击“打开控制台”。

## 5. 更换 API Key

如果你想更换服务商或 API Key，回到首页后点击“更换 API Key”。

重新配置成功后，本工具会再次检查基础状态。确认通过后，可以继续打开控制台使用。

## 6. 遇到问题怎么办

如果安装或配置遇到问题，可以点击底部的“问题排查”。

它会打开安装记录文件夹。普通用户不需要自行理解日志内容，可以把最新的安装记录发给开发者排查。

常见情况：

- 缺少 Node.js 或 npm：请安装 Node.js LTS 版本，npm 通常会一起安装。
- 缺少 Git：可以通过 macOS 命令行工具安装 Git。
- 配置状态为“待确认”：表示可能已有配置，但本工具还不能确认 API Key 是否可用。可以点击“检查现有配置”，或重新配置 API Key。
- 配置状态为“配置异常”：表示检查未通过，建议重新配置 API Key 或查看安装记录。

## 7. 高级选项是什么

页面底部的“高级选项”主要给有经验的用户使用：

- 官方配置向导：打开 OpenClaw 官方完整配置流程，可设置聊天渠道、搜索、技能、自动化等功能。
- 重新检查环境：单独检查这台 Mac 的基础环境。
- 问题排查：查看安装记录。

普通用户首次使用通常只需要按照主流程操作，不需要进入高级选项。
